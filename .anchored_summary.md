## Key Decisions
- **Overlay externalizado, template mantido inline:** Script é JS puro e funciona igual como `.js` externo. Template `clientesTmpl` é HTML estático referenciado por ID — externalizar exigiria fetch + Promise, mudando timing e adicionando complexidade.
- **Sonner CSS removido (não centralizado):** É CSS de biblioteca (Sonner toast), idêntico e gerado pelo bundler. Remover duplicatas é seguro. Centralizar inline não vale risco — em rebuild do bundle pode mudar.
- **Blocos CSS B+C (canvas fixes + clientes-grid patches) não separados:** Em 11/12 páginas estão mesclados em HTML malformado (`<style>canvas...<style>clientes-grid...</style>`). Separar exige reescrita estrutural com risco médio.
- **CSS `.clients-page` (9.7KB) centralizar depois:** Diferença de 2 bytes (whitespace) entre index.html e demais. Potencial de 106 KB de economia, mas requer validação visual.
